## 物理类型

### time
支持自定义格式和标准格式的解析，标准格式支持`rfc_3339` `rfc_2822`
#### 自定义输入格式示例
* `2023-06-01 15:34:39`
* `01/Aug/2023:15:34:39 +0800`
* `Aug 01 15:34:39`
* `2023/6/01 15:34:39`
#### rfc_3393标准输入格式示例
* `2023-06-01T15:34:39−08:00`
* `2023-06-01T15:34:39.001-08:00`
#### rfc_2822标准输入格式示例
* `Tue, 01 Aug 2015 15:34 -0800`
* `01 Aug 2023 15:34:39 +0000`
#### 输出格式
输出格式统一为 `%Y-%m-%d %H:%M:%S`
* `2023-06-01 15:34:39`

### time_ios
iso 标准格式按照`rfc_3339` 标准格式进行支持
#### 输入格式示例
* `2023-06-01T15:34:39−08:00`
* `2023-06-01T15:34:39.001-08:00`
#### 输出格式
输出格式统一为 `%Y-%m-%d %H:%M:%S`
* `2023-06-01 15:34:39`

### time_3339
支持`rfc_3339` 所有标准格式解析
#### 输入格式示例
* `2023-06-01T15:34:39−08:00`
* `2023-06-01T15:34:39.001-08:00`
* `2023-06-01T15:34:39.000031-08:00`
*
#### 输出格式
输出格式统一为 `%Y-%m-%d %H:%M:%S`
* `2023-06-01 15:34:39`

### time_2822
支持`rfc_2822` 所有标准格式解析
#### 输入格式示例
* `Tue, 01 Aug 2023 15:34 -0800`
* `01 Aug 2023 15:34:39 +0000`
* `01 Aug 2023 15:34:39 EST`
#### 输出格式
输出格式统一为 `%Y-%m-%d %H:%M:%S`
* `2023-06-01 15:34:39`

### time_timestamp
时间戳解析
#### 输入格式示例
* `1697618683`
#### 输出格式
输出格式统一为 `%Y-%m-%d %H:%M:%S`
* `2023-06-01 15:34:39`

### kv

支持配置格式：kv、kv\,、kv(type@key)、kv(type@key):new_key\; 等

不支持配置格式：kv(type)

例子：
```txt
log: (log_id = "fdfeeef" status=0, access_time:"2023-11-12 11:12:00";)

rule: (kv, kv(digit@status), kv(time@access_time):recv_time\;)
```
